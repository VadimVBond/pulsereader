# Задача: разработка проекта PulseReader

Проект должен быть реализован с использованием Astro.

Перед началом разработки необходимо внимательно изучить файл PROJECT_CONTEXT.md и дизайн в папке DESIGN_REFERENCE.

Не начинать реализацию, пока архитектура не будет полностью понятна.

---

# ЭТАП 0 — Анализ

Перед разработкой необходимо:

1. Проанализировать файл PROJECT_CONTEXT.md
2. Проверить структуру проекта
3. Изучить дизайн из папки DESIGN_REFERENCE
4. Убедиться, что задачи реализуемы
5. Сообщить о возможных архитектурных улучшениях

После анализа подтвердить готовность к разработке.

---

# ЭТАП 1 — Инициализация проекта

Необходимо:

1. Инициализировать Astro проект
2. Настроить TypeScript
3. Подключить TailwindCSS
4. Создать/проверить базовую структуру папок
5. Настроить глобальные стили

Проверка:
- pnpm install
- pnpm run check
- pnpm run build

На этом этапе страницы реализовывать не нужно.

---

# ЭТАП 2 — Базовая архитектура

Создать:

layouts
- BaseLayout.astro

components
- Header
- Footer
- Container

Все компоненты должны быть переиспользуемыми.

---

# ЭТАП 3 — Компоненты интерфейса

Реализовать:

- ArticleCard
- ThemeToggle

ArticleCard должен отображать:
- заголовок статьи
- краткое описание
- ссылку на статью
- дату публикации (если передана)

ThemeToggle должен:
- переключать светлую и тёмную темы
- сохранять выбранную тему в localStorage
- учитывать системную тему при первом открытии

Проверка:
- pnpm run check
- pnpm run build

---

# ЭТАП 4 — Страницы

Создать страницы:

Главная
- src/pages/index.astro

Страница статьи
- src/pages/article/[slug].astro

Требования:
- динамический маршрут статьи должен корректно работать в static output
- обязательно реализовать getStaticPaths

Проверка:
- pnpm run dev
- открыть /
- открыть /article/<slug>

---

# ЭТАП 5 — Контент

Использовать Astro Content Collections.

Структура:
- src/content/config.ts
- src/content/articles/*.md

Требования:
- статьи хранятся в Markdown
- frontmatter валидируется через schema (zod)
- главная и страница статьи читают данные только из коллекции

---

# ЭТАП 6 — SEO

Добавить:
- meta tags (description, robots, canonical)
- Open Graph / Twitter tags
- sitemap
- rss

Дополнительно:
- дефолтный og:image из public/images для всех страниц
- возможность переопределить image через props layout

---

# ЭТАП 7 — Дизайн-паритет с design.png

Довести UI главной до референса:

1. Внешний shell-контейнер страницы (внутренняя панель с radius и отступами)
2. Блок поиска:
- поле поиска
- чипы фильтров
- кнопка Search
3. KPI-блок с метриками:
- articles found
- sources used
- duplicates removed
4. Контентные секции:
- Trending Articles
- More Stories
5. Карточка статьи v2:
- превью-изображение 16:9
- category label
- заголовок + краткий текст
- нижняя мета-строка (source/time)
6. Иконка поиска в хедере

---

# ЭТАП 8 — Backend RSS Aggregator

Реализовать backend-слой агрегатора новостей.

Структура модуля:
- src/news/sources.ts
- src/news/fetcher.ts
- src/news/parser.ts
- src/news/dedupe.ts
- src/news/pipeline.ts

Требования:
1. Добавить RSS-источники и вывести их как кнопки выбора на главной.
2. Реализовать fetcher RSS с таймаутом и обработкой ошибок.
3. Реализовать парсинг RSS/Atom в единый формат.
4. Нормализовать данные для Astro-страниц (slug/title/excerpt/url/source/date/image).
5. Удалять дубликаты по нормализованному URL/заголовку.
6. Возвращать финальный список статей для страниц Astro и rss.xml.
7. Фильтровать низкокачественные элементы (generic title + пустой excerpt, элементы без изображения и с очень коротким excerpt).

Проверка:
- pnpm run check
- pnpm run build
- pnpm run dev
- открыть /
- открыть /?source=<source-id>
- открыть /article/<slug>
- открыть /rss.xml
- открыть /sitemap.xml
---

# Общие требования

- использовать TypeScript
- не добавлять лишний JavaScript
- следовать архитектуре Astro
- писать чистый и модульный код
- поддерживать светлую и тёмную темы
- избегать полного переписывания файлов без необходимости
